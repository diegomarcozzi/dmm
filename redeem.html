The Challenge

The main challenge was to completely redesign the app for the US market. I have not only helped to redesign the app and the user experience but also the brand, connecting it better with its audiences, including building a more intuitive information architecture and developing a coherent visual language.

Problem solving example: Forced registration vs Guest checkout

Each project has a certain number of problems to solve. Some problems are visible right from the beginning and others arise along the way. The idea is to provide a practical example of my thinking process, problem-solving skills and execution.

Forced registration

The existing layout had no Guest checkout. From previous experiences I already sensed that this could be a problem but I needed data to confirm my suspicions, so I carried out a research comparing the number of Downloads vs Registrations and found out that 23% of users had abandoned the app because they had to create an account. That was the easy part. The hard part was convincing the board that their successful app needed a major change: Guest checkout.


My arguments:

It can reduce abandonment rates.
(23% had abandoned a site just because they had to create an account).

It’s much easier for mobile users.
Each extra form field or step can be a chore for mobile shoppers.

It gets shoppers straight into payment forms.
Without registration, potential users aren’t presented with a point that makes them stop and think about continuing.

It makes checkout seem like less work.
In reality, the same details are generally entered while users complete guest checkout, but the flow makes it seem easier.

You can still provide a registration option.
We can still add the option to create an account at the other end of the checkout process.

Ideation

I wanted a subtle, imperceptible, easy, uninterrupted, smooth checkout process. I checked with my team on each item needed to make sure we were on the same page. After a few drawings I came up with a succesful version of the User Flow.

Once the User Flow was ready, I began to work on the Wireframe. This took a bit longer. I usually like to play with a blackboard or notepad before jumping into XD or Figma.

Final version

The user no longer needs to register before using the application. Forced registration has been replaced by a subtle multi-step process that begins once the user selects a service or product and presses the "Buy" button.

The call-to-action button has four different states that guide the user through the process.


1. Add Contact Information
The users have to enter their email address to continue with the process.

2. Add Payment Method
This button is displayed once the email address has been entered.

3. Place Order
This button is only shown if the user has chosen to pay with a credit/debit card and has already entered their data correctly.

4. Continue to Paypal
This button is only shown if the user has chosen to pay with Paypal.

Sign in and login screens were simplified to prevent overloading the user with unnecessary information that could generate interference and its consequent interruption and/or abandonment of the registration process.
